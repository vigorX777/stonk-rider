# 韭菜骑士 / Leek Knight 项目规则

## 项目定位

韭菜骑士 / Leek Knight 是一个把真实 A 股日线转换为摩托车赛道的纯前端物理游戏。

## 修改规则

1. 游戏运行时不得依赖行情接口或后端服务。
2. 收益计算必须来自原始行情，不得使用经过坡度修正后的物理坐标。
3. 赛道转换逻辑保持为可测试纯函数；Phaser 场景只负责物理、渲染和输入。
4. 涨跌颜色遵循 A 股习惯：红涨绿跌，同时提供符号或文字提示。
5. 架构、数据管线或玩法规则变化时，同步更新对应项目文档和 `CHANGELOG.md`。
6. 交付前至少运行 `npm test`、`npm run build`，并进行浏览器实玩验证。

## 目录职责

- `src/data/`：浏览器行情快照读取。
- `src/game/`：收益、赛道、音频和 Phaser 游戏循环。
- `public/data/stocks/`：构建期生成的固定行情快照。
- `scripts/`：行情抓取与快照生成工具。
- `tests/`：纯逻辑回归测试。
